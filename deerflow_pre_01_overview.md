# DeerFlow 第一章：整体架构概览（小明的第一次体验）

> **目标**：让非技术背景的人也能理解 DeerFlow 是怎么工作的  
> **风格**：用"小明"的用户故事，解释"用户做了什么 → 系统发生了什么"  
> **章节**：第 1 章，共 5 章

---

## 1.1 小明打开 DeerFlow 网页

### 场景：在浏览器输入 `localhost:2026`

**小明看到了什么？**

一个漂亮的聊天界面，有输入框、发送按钮、设置面板。

**背后发生了什么？**

```
小明输入 localhost:2026
    ↓
Nginx（门卫）站在 2026 号门口收到请求
    ↓
Nginx 查看路由表："这是访问网页，不是 API 请求"
    ↓
转发给 Frontend（前端应用，住在 3000 号房间）
    ↓
Frontend 返回聊天界面
    ↓
小明看到了网页
```

**通俗解释**：

想象 DeerFlow 是一栋大楼：
- **Nginx** = 大楼门卫，站在 2026 号门口
- **Frontend** = 前台接待处（3000 号房间），负责展示界面
- **门卫有一张路由表**：

| 访客请求 | 送到哪里 |
|---------|---------|
| 看网页（如 `/chat`） | Frontend（3000） |
| 调用 AI（`/api/langgraph/*`） | LangGraph Server（2024） |
| 其他 API（`/api/*`） | Gateway API（8001） |

## 1.2 小明发送第一条消息

### 场景：输入"帮我研究 AI 趋势"，点击发送

**小明看到了什么？**

AI 开始回复，一行一行显示出来，像是在实时打字。

**背后发生了什么？**

```
小明输入"帮我研究 AI 趋势"
    ↓
Frontend 把消息打包，发送到 /api/langgraph
    ↓
Nginx 识别："这是 AI 请求，要找大脑"
    ↓
转发给 LangGraph Server（2024 号房间）
    ↓
LangGraph Server 启动 Lead Agent（总指挥）
    ↓
Lead Agent 思考："这个任务需要做什么？"
    ↓
分解任务 → 调用工具 → 可能创建子代理 → 收集结果
    ↓
通过 SSE（实时流）返回结果给 Frontend
    ↓
Frontend 一行一行显示，小明看到实时回复
```

**关键角色介绍**：

| 角色 | 职责 | 类比 |
|------|------|------|
| **LangGraph Server** | AI 大脑，负责思考 | 公司 CEO |
| **Lead Agent** | 总指挥，协调任务 | 项目经理 |
| **Middleware** | 中间处理环节 | 各部门审批 |
| **Sandbox** | 隔离的工作环境 | 独立办公室 |

## 1.3 小明上传了一份 PDF

### 场景：点击"上传文件"，选择一份研究报告

**小明看到了什么？**

文件显示在聊天界面，AI 说"我已阅读这份报告"。

**背后发生了什么？**

```
小明点击"上传 PDF"
    ↓
Frontend 调用 POST /api/threads/{id}/uploads
    ↓
Nginx 转发给 Gateway（8001 号房间，后勤部）
    ↓
Gateway 接收文件，保存到指定位置
    ↓
检测到是 PDF，自动转换成 Markdown 文本
    ↓
返回文件信息给 Frontend
    ↓
下次对话时，UploadsMiddleware 自动告诉 AI：
    "用户上传了这份文件，内容如下..."
    ↓
AI 基于文件内容回答问题
```

**文件处理流程**：

| 步骤 | 做什么 | 结果 |
|------|--------|------|
| 1. 接收 | Gateway 接收上传的文件 | 保存到 uploads 目录 |
| 2. 转换 | 调用 markitdown 工具 | PDF → Markdown |
| 3. 存储 | 保存转换后的文本 | 方便 AI 阅读 |
| 4. 注入 | UploadsMiddleware 工作 | AI 能看到文件内容 |

## 1.4 小明想切换 AI 模型

### 场景：打开设置面板，看到模型列表

**小明看到了什么？**

下拉框显示：GPT-4o、Claude 3 Opus、DeepSeek V3...

**背后发生了什么？**

```
小明打开设置面板
    ↓
Frontend 调用 GET /api/models
    ↓
Nginx 转发给 Gateway（8001）
    ↓
Gateway 读取 config.yaml 配置文件
    ↓
返回模型列表：[
    {"name": "gpt-4o", "display_name": "GPT-4o"},
    {"name": "claude-3-opus", "display_name": "Claude 3 Opus"},
    {"name": "deepseek-v3", "display_name": "DeepSeek V3"}
]
    ↓
Frontend 展示在下拉框中
    ↓
小明选择"DeepSeek V3"
    ↓
下次对话时，使用 DeepSeek V3 模型
```

**注意**：Gateway 只是"读配置"，真正的 AI 推理在 LangGraph 里进行。

## 1.6 核心概念速查表

| 术语 | 通俗解释 | 类比 |
|------|---------|------|
| **Nginx** | 门卫，负责分发请求 | 大楼前台 |
| **Frontend** | 前端界面，用户看到的网页 | 接待大厅 |
| **LangGraph Server** | AI 大脑，负责思考和推理 | CEO 办公室 |
| **Gateway** | 后勤部，处理文件/配置等非AI事务 | 行政部 |
| **Lead Agent** | 总指挥，协调任务执行 | 项目经理 |
| **Thread** | 对话线程，保持上下文记忆 | 项目档案 |
| **Sandbox** | 隔离的工作环境 | 独立办公室 |

## 本章小结

**小明的一次完整体验**：
1. 打开网页 → Nginx 转发给 Frontend → 看到界面
2. 发送消息 → Nginx 转发给 LangGraph → AI 思考回复
3. 上传文件 → Gateway 处理保存 → AI 能读取内容
4. 切换模型 → Gateway 读取配置 → 下次使用新模型

**记住这张图**：
- **2026** = Nginx 门卫
- **3000** = Frontend 前台
- **2024** = LangGraph AI 大脑
- **8001** = Gateway 后勤部
